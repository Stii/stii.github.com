Wordpress seems to dominate publishing on the internet these days. While it is one of the greatest open source and free platforms for publishing I’ve grown tired of it. I do not have to be able to publish anywhere anytime. That is what Twitter/Facebook is for these days. I just want to compile and publish a bunch of thoughts. That is enough for me.

Secondly, I love tinkering. I cannot think of anything worse than spending my free time plumbing PHP. That is just a horrible thought. I want to learn new things. Make code better. Improve my skills and expand my knowledge. What I do love to tinker with in my spare time is Haskell. It is not likely that I would ever land in a job that would require Haskell skills. Some day when I feel I’ve mastered Haskell, I may use it in a project other than this little site, but there is much to learn still. For now, I’ll just hack around with it.

Which brings me to my point. Static site generators. They’re just what the doctor ordered. So what exactly is it? It is exactly that which the name implies. It generates static sites. Now you would probably think, why not simply then create a HTML site. Why do you need software to generate the site? Well, you wouldn’t want to maintain hundreds of HTML pages now, would you? Basically, the generator allows you to do offline, what your Wordpress template does online. It allows you to reuse a template for your pages and just replace the content that changes. The difference is that it does so before you actually publish anything. What makes this really awesome is that it only uploads static HTML to your host. So the site would be lightweight and simple. No need for databases or any specific programming language required. You have what you need. Guaranteed!It allows me to write my posts using markdown, then compile the site, generating all the new links and reusing the standard template without me having to maintain more than one template page.

There are a number of site generators in almost every flavour of language. Even PHP if you have to. I’ve selected to go with Hakyll which is in Haskell. It is based on Nanoc which is a Ruby generator, but borrows its name from Jekyll (another Ruby generator). I publish it to Github or more specifically, Github:Pages. It is free and easy. I simply push the site to a repo and viola!

Besides the ease and simplicity, it is quite satisfactory. Writing is fun again. For me anyway. I won’t recommend it to everyone. If you are not technical or all you want to do is run a publishing site (like a blog) this is probably not for you. If you need a database or session handling or some custom development, this is definitey not for you. If all you want is a little site where you can write while you hack on a piece of software, this is for you.

Built using some of my favorite tools: Hakyll and Vim.
